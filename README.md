# 🧩 Balda Helper

**Balda Helper** — это desktop-приложение на Python для поиска лучших ходов в игре **«Балда»**.

Программа помогает быстро находить возможные слова на текущем поле, показывает, куда можно поставить новую букву, подсвечивает путь чтения слова, учитывает уже использованные слова и позволяет удобнее вести игру.

Приложение предназначено не для самой игры как отдельной игровой системы, а именно для **подсказок и анализа текущей партии**.

## 📸 Как работает приложение

### 1. Пользователь вводит поле

На поле вводятся уже существующие буквы игры.

Например:

```text
. . . . .
. . о . .
. . к . .
. . н . .
. . . . .
```

Пустые клетки остаются пустыми.

Можно выбрать:

- размер поля;
- минимальную длину слова;
- лимит подсказок;
- использовать диагонали или нет.

---

### 2. Программа ищет возможные ходы

После нажатия **«Найти подсказки»** приложение перебирает возможные варианты постановки одной новой буквы и ищет слова, которые можно составить на текущем поле.

В подсказках отображаются:

| Колонка | Значение |
|---|---|
| `Слово` | найденное слово |
| `Букв` | длина слова |
| `Варианты` | куда и какую букву можно поставить |
| `?` | определение слова |
| `Убрать` | скрыть слово из подсказок |

Одно слово показывается один раз, а все варианты постановки буквы находятся внутри него.

Пример:

```text
кот      3      2 варианта
                «к» в (2, 1)
                «т» в (2, 3)
```

---

### 3. Наведение показывает путь

Если навести мышку на конкретный вариант постановки буквы, приложение подсветит путь чтения слова прямо на поле.

Например:

```text
к → о → т
```

Новая буква выделяется отдельным цветом.

---

### 4. Клик по варианту применяет ход

Если кликнуть по варианту, программа:

1. ставит новую букву на поле;
2. добавляет слово в использованные;
3. убирает слово из подсказок;
4. пересчитывает новые подсказки уже по обновлённому полю.

---

### 5. Использованные слова

Использованные слова отображаются в отдельной области в виде кликабельных плашек.

```text
[ кот ] [ окно ] [ дом ] [ река ]
```

Если кликнуть по использованному слову, оно удаляется из списка.

Если это слово добавляло букву на поле, эта буква также удаляется.

## 🧠 Почему поиск быстрый

Главная оптимизация — использование **Trie**, или префиксного дерева.

Обычный медленный подход выглядел бы так:

```text
для каждого слова в словаре
    проверить, можно ли составить его на поле
```

Это плохо работает, если словарь большой.

В Balda Helper используется другой подход:

```text
идём по полю
собираем буквы
после каждой буквы проверяем:
    существует ли в словаре слово с таким префиксом?
если нет — сразу останавливаем ветку поиска
```

Например, если во время обхода получилось:

```text
кщ
```

и в словаре нет слов на `кщ`, программа сразу прекращает этот путь.

Это сильно уменьшает количество проверок.

---

## 🌳 Trie / префиксное дерево

Trie позволяет быстро отвечать на вопросы:

```text
есть ли слово "кот"?
есть ли слова, начинающиеся на "ко"?
можно ли продолжать путь по полю?
```

Пример:

```text
к
└── о
    ├── т
    └── н
        └── ь
```

Так программа быстро понимает, что:

```text
ко
```

можно продолжать, а, например:

```text
кщ
```

нельзя.

---

## 🔁 Обратный Trie

Для подсказок важно, что новое слово в «Балде» обязательно проходит через новую букву.

Слово может выглядеть так:

```text
левая часть + новая буква + правая часть
```

Например:

```text
к + о + т
```

Чтобы искать такие варианты быстрее, используется не только обычный Trie, но и **обратный Trie**.

Обычный Trie помогает продолжать слово вправо:

```text
к → о → т
```

Обратный Trie помогает искать начало слова назад:

```text
т → о → к
```

Это позволяет строить слово вокруг новой буквы, а не запускать полный поиск с каждой клетки поля.

---

## 🧮 Сортировка подсказок

Подсказки сортируются так:

1. сначала идут более длинные слова;
2. среди слов одинаковой длины выше идут слова из более частых букв;
3. затем применяется алфавитная сортировка.

Частоты букв не задаются вручную.

Они автоматически считаются по загруженному словарю.

То есть приложение само анализирует словарь и понимает, какие буквы встречаются чаще.

Это помогает опускать ниже слова с редкими буквами, если длина слов одинаковая.

## 📚 Словарь

Приложение использует JSON-словарь:

```text
data/balda_nouns.json
```

В словарь попадают существительные русского языка.

При подготовке словаря исключаются:

```python
BAD_GRAMMEMES = {
    "Name",  # имя
    "Surn",  # фамилия
    "Patr",  # отчество
    "Geox",  # географическое название
    "Orgn",  # организация
    "Trad",  # торговая марка
    "Abbr",  # аббревиатура
    "Init",  # инициалы
}
```

Также буква `ё` не учитывается отдельно:

```text
ёлка → елка
```

---

## 🧩 Пользовательские слова

Если другой игрок назвал слово, которого нет в словаре, приложение спросит:

```text
Слово не найдено.
Вы уверены, что оно правильное?
```

Если пользователь подтверждает, слово добавляется в словарь текущей сессии.

Важно:

> Такое слово добавляется только в память приложения.  
> После перезапуска оно не сохранится в `balda_nouns.json`, если специально не реализовать сохранение пользовательских слов.

## ❓ Определения слов

В подсказках есть колонка `?`.

При наведении на неё приложение загружает определение слова через отдельный backend-сервис:

```text
services/definitions.py
```

Интерфейс не занимается поиском определений напрямую.

Он только запрашивает определение у сервиса и показывает красивое всплывающее окно.

Это сделано специально, чтобы не смешивать UI и backend-логику.

## 🗂 Основные модули

### `app.py`

Точка входа в приложение.

Создаёт `QApplication`, открывает главное окно и запускает event loop.

---

### `solver/trie.py`

Реализация Trie.

Используется для:

- проверки слов;
- проверки префиксов;
- быстрого обхода поля;
- отсечения невозможных путей.

---

### `solver/dictionary.py`

Загружает JSON-словарь и строит:

- `set` всех слов;
- обычный Trie;
- обратный Trie;
- максимальную длину слова.

---

### `solver/board.py`

Описывает игровое поле.

Отвечает за:

- хранение букв;
- нормализацию ввода;
- проверку координат;
- кэш соседних клеток;
- работу с диагоналями.

---

### `solver/hints.py`

Главный модуль поиска подсказок.

Он:

- перебирает пустые клетки;
- пробует возможные буквы;
- строит слова вокруг новой буквы;
- возвращает найденные варианты с координатами пути.

---

### `services/definitions.py`

Backend-сервис для получения определений слов.

UI не знает деталей поиска определения.

Он просто вызывает сервис.

---

### `ui/main_window.py`

Главное окно приложения.

Отвечает за:

- сборку интерфейса;
- обработку кнопок;
- запуск поиска;
- работу с использованными словами;
- отображение подсказок;
- взаимодействие между UI и backend-сервисами.

---

### `ui/widgets/board_widget.py`

Кастомное игровое поле.

Отвечает за:

- клетки поля;
- ввод букв;
- навигацию стрелками;
- клики по ячейкам;
- подсветку пути;
- стрелки между клетками.

---

### `ui/widgets/used_words_cloud.py`

Виджет использованных слов.

Показывает слова как кликабельные плашки:

```text
[ кот ] [ дом ] [ окно ]
```

При наведении плашка подсвечивается, а курсор становится рукой.

---

### `ui/widgets/definition_popup.py`

Красивое всплывающее окно с определением слова.

---

### `ui/threads.py`

Потоки для тяжёлых операций:

- поиск подсказок;
- загрузка определения слова.

Это нужно, чтобы интерфейс не зависал.

---

### `ui/styles.py`

Общий stylesheet приложения.

---

### `utils/resources.py`

Помогает находить файлы ресурсов как при обычном запуске, так и внутри PyInstaller-сборки.

Например, словарь:


## 🧭 Работа с ресурсами в PyInstaller

Чтобы приложение находило словарь после сборки, используется helper:

```python
from utils.resources import resource_path

DICTIONARY_PATH = resource_path("data/balda_nouns.json")
```

Пример `utils/resources.py`:

```python
from __future__ import annotations

import sys
from pathlib import Path


def resource_path(relative_path: str) -> Path:
    base_path = getattr(sys, "_MEIPASS", None)

    if base_path is not None:
        return Path(base_path) / relative_path

    return Path(relative_path)
```

Это позволяет программе находить `data/balda_nouns.json` и при обычном запуске, и внутри `.exe`.

## 🧹 Какие файлы можно удалять после сборки

После PyInstaller появляются:

```text
build/
dist/
BaldaHelper.spec
```

Можно удалить:

```text
build/
```

Это временные файлы.

Можно удалить:

```text
BaldaHelper.spec
```

Но лучше оставить, если планируешь пересобирать приложение с теми же настройками.

Нельзя удалять:

```text
dist/
```

если именно там лежит готовая программа.

## 💡 Как пользоваться приложением

1. Запусти приложение.

## 🧩 Добавление слова другого игрока

Если другой игрок назвал слово:

1. нажми **«Добавить новое слово»**;
2. введи слово;
3. если слова нет в словаре, подтверди, что оно правильное;
4. кликни по пустой клетке, куда была поставлена новая буква;
5. если программа не может определить букву автоматически, введи букву вручную.

После этого слово будет добавлено в использованные, а поле обновится.


## 🧠 Почему приложение полезно

В «Балде» часто сложно быстро оценить все возможные варианты, особенно на больших полях.

Balda Helper помогает:

- увидеть длинные слова;
- не забыть уже использованные слова;
- быстро проверить разные варианты;
- понять направление чтения слова;
- найти слова, которые человек мог пропустить;
- вести игру аккуратнее и быстрее.
